# UI 右键菜单优化任务书

本文档记录当前 UI 中适合新增鼠标右键菜单的场景、菜单项设计和实现注意事项。目标是让列表、表格、日志、历史等高频对象可以就地操作，而不是必须先选中再去找顶部按钮或“更多”按钮。

## 总体原则

1. 右键菜单是快捷入口，不替代现有按钮。
2. 菜单项必须与当前行/单元格上下文相关。
3. 右键某一行时，应先选中该行，再弹菜单，避免操作对象不明确。
4. 危险操作保留确认弹框，例如删除流程、删除步骤、删除环境、删除变量。
5. 不要把所有功能都塞进右键菜单；右键菜单只放当前对象最自然的操作。
6. 菜单项顺序建议：
   - 打开/编辑/运行等主操作
   - 复制/导出/复制文本等辅助操作
   - 分隔线
   - 删除等危险操作

## Fyne 实现注意事项

Fyne 的 `widget.List` / `widget.Table` 默认不直接提供完整的行级右键菜单 API。后续 Agent 需要按当前 Fyne 版本选择合适方案。

建议方案：

1. 为列表 item 或表格 cell 包装自定义组件，实现 secondary tap。
2. 常见接口方向：
   - `fyne.SecondaryTappable`
   - `TappedSecondary(*fyne.PointEvent)`
3. 在 `TappedSecondary` 中：
   - 先更新当前选中行/对象。
   - 再调用 `widget.ShowPopUpMenuAtPosition(...)` 或 `widget.ShowPopUpMenuAtRelativePosition(...)`。
4. 若当前 Fyne 版本对 `List` item 的 secondary tap 支持有限，可先在 row 内放一个轻量“更多”图标作为备用，但优先目标仍是右键。
5. 菜单弹出位置应使用鼠标位置，不要固定到工具栏按钮位置。

建议新增通用 helper：

```text
internal/ui/context_menu.go
```

可包含：

```go
type secondaryTapLabel struct { ... }
func showContextMenu(win fyne.Window, pos fyne.Position, menu *fyne.Menu)
func menuItemCopy(label string, value func() string)
```

## P0-1. 流程库列表右键菜单

### 当前位置

```text
internal/ui/flow_library.go
```

当前已有：

- 左侧流程列表 `widget.List`
- 顶部按钮：新建、保存、更多
- “更多”菜单中已有新建、从模板创建、导入、导出、复制、删除

### 问题

用户对某个流程的操作应能直接在流程行上右键完成。现在需要先左键选中，再找“更多”，上下文不够直接。

### 右键目标

流程列表的每个流程行。

### 菜单项建议

```text
打开流程
运行此流程
从此步骤运行（如果能结合当前步骤则可后续做）
复制流程
导出流程
复制流程名称
复制流程 ID
删除流程
```

### 行为要求

- 右键流程行时先选中该流程并加载完整 flow。
- `打开流程` 等同于选择该流程。
- `运行此流程` 调用 `runCurrentFlow()`，但必须先完成流程选择。
- `复制流程` 调用现有 `onFlowClone(...)`。
- `导出流程` 调用现有 `exportFlow()`，但要确保导出的是右键所在流程。
- `删除流程` 调用现有 `onFlowDelete(...)`。
- 复制类操作写入剪贴板，并在日志中提示。

### 验收标准

- 右键任意流程行出现菜单。
- 同名流程也能正确操作右键所在行。
- 删除仍有确认弹框。

## P0-2. 步骤表格右键菜单

### 当前位置

```text
internal/ui/step_table.go
```

当前已有：

- 步骤表格 `widget.Table`
- 选中步骤后显示操作条：复制、删除、上移、下移

### 问题

步骤编排是高频编辑区域，右键行操作会明显提升效率。

### 右键目标

步骤表格的任意数据行。右键任意 cell 都应以该 cell 所在 row 为操作对象。

### 菜单项建议

```text
编辑步骤属性
从此步骤运行
单步执行到此处（可选，后续增强）
复制步骤
删除步骤
上移
下移
启用/禁用
复制步骤名称
复制 XPath/目标
复制输入内容
```

### 行为要求

- 右键 cell 时先选中该 row。
- `编辑步骤属性` 加载右侧属性面板。
- `从此步骤运行` 应设置当前选中行后调用 `runCurrentFlow()`，现有 `selectedStartStep()` 已支持从选中步骤开始。
- `复制步骤` 调用现有 `copyStep()`。
- `删除步骤` 调用现有 `deleteStep()`，建议增加确认弹框，避免误删。
- `启用/禁用` 切换 `Step.Enabled` 并标记 dirty。
- 复制 XPath/输入内容时要尊重敏感输入：
  - 如果 `MaskInLogs` 为 true，右键复制输入内容前需要确认，或不提供该项。

### 验收标准

- 右键任意步骤行出现菜单。
- 从第 N 行右键运行时，从第 N 步开始执行。
- 禁用步骤后表格状态刷新，流程标记为未保存。

## P0-3. 环境列表右键菜单

### 当前位置

```text
internal/ui/env_panel.go
```

当前已有：

- 环境列表 `widget.List`
- 环境操作按钮菜单：重命名、复制、删除、设为当前

### 右键目标

环境配置 Tab 左侧环境列表的每个环境行。

### 菜单项建议

```text
设为当前环境
重命名 / 说明
复制环境
导出环境配置
复制环境名称
复制环境 ID
删除环境
```

### 行为要求

- 右键环境行时先选中该环境。
- `设为当前环境` 调用 `showSetActiveEnvDialog()` 或直接 `envRepo.SetActive(...)`。
- `导出环境配置` 当前如果只能导出全部环境，应命名为 `导出全部环境配置`；如果后续支持单环境导出，再命名 `导出此环境`。
- 删除环境仍需确认。

### 验收标准

- 右键环境行出现菜单。
- 设为当前后，全局环境下拉同步刷新。
- 删除当前环境后有合理 fallback。

## P0-4. 环境变量表右键菜单

### 当前位置

```text
internal/ui/env_panel.go
```

当前已有：

- 环境变量表 `widget.TableWithHeaders`
- 双击/选择当前行会编辑变量（当前 `OnSelected` 直接 `showEditVarDialog()`）
- 变量操作按钮菜单：编辑变量、删除变量

### 问题

环境变量表是典型需要右键操作的表格，尤其是复制 key/value、编辑、删除。

### 右键目标

环境变量表任意数据 cell。

### 菜单项建议

```text
编辑变量
复制变量名
复制变量值
复制 env 引用：${env:KEY}
切换敏感标记
删除变量
```

### 行为要求

- 右键 cell 时先选中对应变量。
- `复制变量值`：
  - 若变量为敏感变量，默认不显示明文，但可以提供复制。
  - 复制敏感变量值前应弹确认，文案说明会复制明文到剪贴板。
- `复制 env 引用` 将 `${env:KEY}` 写入剪贴板，便于用户在流程步骤里使用。
- `切换敏感标记` 直接修改 `IsSecret` 并保存。
- 删除变量仍需确认。

### 验收标准

- 右键变量行出现菜单。
- 可以快速复制 `${env:KEY}`。
- 复制敏感变量值有确认。

## P0-5. 运行日志右键菜单

### 当前位置

```text
internal/ui/run_panel.go
```

当前已有：

- 清空日志按钮
- 复制日志按钮
- 打开产物目录按钮
- 日志内容为 `canvas.Text` 添加到 `logBox`

### 问题

日志是排查问题的核心区域。用户通常会右键复制某一行、复制全部、清空、打开产物目录。

### 右键目标

运行详情页的日志区域。最好支持：

- 右键单条日志行
- 右键日志空白区域

### 菜单项建议

```text
复制当前日志行
复制全部日志
清空日志
打开产物目录
复制产物目录路径
```

### 行为要求

- 当前日志行需要自定义可右键的 log item，而不是裸 `canvas.Text`。
- 如果无法定位具体行，至少在日志区域右键提供：
  - 复制全部日志
  - 清空日志
  - 打开产物目录
- 清空日志可直接执行或二次确认；如果日志很多，建议确认。

### 验收标准

- 右键日志区域出现菜单。
- 可复制当前行或全部日志。
- 清空日志和顶部按钮行为一致。

## P0-6. 历史记录列表右键菜单

### 当前位置

```text
internal/ui/history_panel.go
```

当前已有：

- 历史列表 `widget.List`
- 环境筛选、状态筛选

### 问题

历史记录适合右键查看详情、复制结果、打开产物目录、按同环境重跑等。

### 右键目标

历史记录列表的每一行。

### 菜单项建议

```text
查看详情
复制运行摘要
复制运行 ID
打开产物目录
按此环境重新运行
筛选此环境
```

### 行为要求

- 右键历史行时先选中该记录。
- `复制运行摘要` 包含：
  - 时间
  - 状态
  - 成功/失败/跳过数量
  - 环境
  - 耗时
- `打开产物目录` 如果当前 RunResult 没有目录字段，需要从 step result 的截图/HTML 路径推导；没有产物时禁用。
- `按此环境重新运行` 切换全局环境后调用运行当前流程，需确认当前流程与历史记录 flow 是否一致。

### 验收标准

- 右键历史记录出现菜单。
- 可复制运行摘要。
- 有产物时可打开目录。

## 推荐实施顺序

1. 先新增通用右键菜单基础组件/helper。
2. 实现流程库列表右键菜单。
3. 实现步骤表格右键菜单。
4. 实现环境列表和环境变量表右键菜单。
5. 实现运行日志右键菜单。
6. 实现历史记录右键菜单。
7. 后续再做模板选择器、步骤属性输入框、产物路径、设置页路径输入框。

## 手工验收清单

- 右键流程行：可打开、运行、复制、导出、删除。
- 右键步骤行：可编辑、从此步骤运行、复制、删除、上移、下移、启用/禁用。
- 右键环境行：可设为当前、重命名、复制、删除。
- 右键变量行：可编辑、复制 key、复制 `${env:KEY}`、删除。
- 右键日志区域：可复制日志、清空日志、打开产物目录。
- 右键历史记录：可复制摘要、复制运行 ID、打开产物目录。
- 危险操作仍有确认弹框。
- 右键操作不会误操作未选中的其他行。
- 右键菜单位置跟随鼠标，不固定到页面左上角或工具栏按钮。

## 注意事项

- 不要移除现有按钮和“更多”菜单；右键是补充入口。
- 不要在右键菜单中暴露无确认的敏感变量明文复制。
- 不要让右键菜单破坏 `OnSelected` 现有逻辑，尤其是流程需要加载完整 steps。
- 不要提交 `data/`、`logs/`、`chrome/`、`go-chrome` 二进制或运行产物。
