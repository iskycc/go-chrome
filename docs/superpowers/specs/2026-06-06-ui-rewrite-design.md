# Go-Chrome UI 全量重写设计文档

**日期**: 2026-06-06  
**范围**: UI 阶段 A/B/C/D（状态栏、动态表单、运行诊断、新手引导）  
**方案**: 方案二 — 全量重写 UI 布局

---

## 1. 设计目标

- 新用户不看 README 也能按区域编号完成一次流程创建和运行。
- 所有用户可见文案为中文。
- 用户能明确判断按钮属于流程、步骤、属性还是运行。
- UI 不暴露 `clear_and_input`、`assert_text` 等内部枚举。
- 输入错误能在字段附近看到原因。
- 运行失败后能快速定位失败步骤、错误原因、截图和 HTML 快照。
- JSON schema 保持兼容，不因中文 UI 改动改变持久化结构。

## 3. 阶段 A：顶部状态栏与保存状态

### 3.1 状态栏结构

使用水平 `container.NewHBox`，分成 5 个状态胶囊，每个胶囊包含图标 + 文本 + 颜色圆点：

| 胶囊 | 内容 | 状态值 | 颜色 |
|------|------|--------|------|
| 产品 | Go Chrome | — | 灰色 `#9e9e9e` |
| 当前流程 | 未选择流程 / 流程名 | — | 灰色 / 蓝色 `#1a73e8` |
| 保存状态 | 未修改 / 有未保存修改 / 保存中 / 已保存 / 保存失败 | 黄色 `#f9a825` / 蓝色 / 绿色 `#4caf50` / 红色 `#e53935` |
| Chrome 状态 | 未安装 / 已安装 / 下载中 / 启动中 / 已启动 / 启动失败 | 灰色 / 绿色 / 蓝色 / 红色 |
| 运行状态 | 空闲 / 运行中 3/12 / 已完成 / 失败于第 5 步 | 灰色 / 蓝色 / 绿色 / 红色 |

### 3.2 状态枚举定义

```go
// internal/ui/status.go
type SaveStatus int
const (
    SaveUnmodified SaveStatus = iota
    SaveDirty
    SaveSaving
    SaveSuccess
    SaveFailed
)

type ChromeStatus int
const (
    ChromeNotInstalled ChromeStatus = iota
    ChromeInstalled
    ChromeDownloading
    ChromeStarting
    ChromeRunning
    ChromeStartFailed
)

type RunStatus int
const (
    RunIdle RunStatus = iota
    RunRunning
    RunCompleted
    RunFailed
)
```

### 3.3 保存状态跟踪

在 `ui.App` 中新增 `dirty bool` 和 `saveStatus SaveStatus`。

触发 `markDirty()` 的事件：
- 流程属性（名称、描述、标签）的 `OnChanged`。
- 步骤表格中启用/禁用切换。
- 步骤属性区点击「应用到当前步骤」。
- 新增 / 复制 / 删除 / 上移 / 下移步骤。

触发 `markClean()` 的事件：
- 保存成功。
- 切换到已保存的其他流程（且当前流程已保存）。

### 3.4 Chrome 状态轮询

`browser.Manager` 新增 `Status() ChromeStatus` 方法。

UI 启动一个 1 秒一次的 `time.Ticker`，在 `fyne.Do` 中更新状态胶囊。运行阶段临时提高频率到 200ms。

## 4. 阶段 B：步骤表格与动态表单

### 4.1 步骤表格

将 `widget.List` 替换为 `widget.Table`，列定义：

```text
序号 | 状态 | 启用 | 步骤名称 | 操作类型 | 目标摘要 | 输入摘要 | 等待 | 失败处理
```

单元格截断规则：
- 步骤名称：> 14 字符截断为 `...`。
- 目标摘要 / 输入摘要：> 20 字符截断；若开启日志脱敏，输入摘要显示 `***`。
- 等待：显示毫秒数，如 `1000ms`。

样式：
- 使用 `widget.Table` 的自定义 `CreateCell` / `UpdateCell` 渲染每行内容，单元格内使用 `canvas.Text` 控制颜色和截断。
- 禁用行文本颜色变浅（`color.NRGBA{R: 0, G: 0, B: 0, A: 120}`）。
- 选中行背景色通过 `Table.OnSelected` 刷新实现高亮。

### 4.2 新增步骤流程

点击 `新增步骤` 后弹出类型选择对话框：

```go
widget.NewSelect(stepTypeOptions, func(selected string) {
    // 用户选择后，在当前选中步骤后插入
})
```

步骤默认名称规则：`{操作类型中文名}步骤`，例如 `点击元素步骤`。

### 4.3 动态表单字段矩阵

右侧 `stepPropertyPanel` 使用 `container.NewVBox`，根据当前步骤类型动态生成表单。显示/隐藏规则：

| 步骤类型 | 显示字段 |
|----------|---------|
| 打开网址 | 步骤名称、URL、执行后等待、超时、失败处理、备注 |
| 点击元素 | 步骤名称、XPath、执行前等待、执行后等待、超时、失败处理、备注 |
| 输入文本 | 步骤名称、XPath、输入内容、模板预览按钮、日志脱敏、等待、超时、失败处理、备注 |
| 清空后输入 | 同输入文本 |
| 等待元素出现 | 步骤名称、XPath、超时、失败处理、备注 |
| 等待元素可见 | 步骤名称、XPath、超时、失败处理、备注 |
| 固定等待 | 步骤名称、等待毫秒数、备注 |
| 获取元素文本 | 步骤名称、XPath、超时、失败处理、备注 |
| 断言元素存在 | 步骤名称、XPath、超时、失败处理、备注 |
| 断言文本包含 | 步骤名称、XPath、期望文本、超时、失败处理、备注 |
| 页面截图 | 步骤名称、截图备注、失败处理、备注 |

### 4.4 字段级实时校验

每个输入控件旁放置一个 `widget.Label`，初始隐藏（`Hide()`），校验失败时显示红色错误文本。

校验规则矩阵：

| 字段 | 校验规则 | 错误文案 |
|------|---------|---------|
| 步骤名称 | 非空 | "步骤名称不能为空" |
| 步骤名称 | 当前流程内唯一 | "步骤名称已存在" |
| URL | 非空 | "网址不能为空" |
| URL | 以 `http://` 或 `https://` 开头 | "网址必须以 http:// 或 https:// 开头" |
| XPath | 非空（元素类操作） | "XPath 不能为空" |
| 期望文本 | 非空（断言文本包含） | "期望文本不能为空" |
| 等待 / 超时 | 非负整数 | "必须为非负整数" |
| 等待毫秒数 | 非负整数 | "必须为非负整数" |
| 输入内容 | 通过 `template.Validate()` | 返回模板引擎的具体错误 |

### 4.5 切换前未保存提示

切换流程、关闭窗口、导入流程前，如果 `dirty == true`，弹出三按钮对话框：

```text
标题：未保存的修改
内容：当前流程 "XXX" 有未保存的修改，是否保存？
按钮：[保存并切换] [放弃修改] [取消]
```

## 5. 阶段 C：运行诊断闭环

### 5.1 运行区布局

底部 `runPanel` 使用 `container.NewBorder`：

```text
┌─────────────────────────────────────────────────────────┐
│ 进度条 (5/12)  [启动浏览器] [运行] [单步] [停止]          │
├──────────────────────────────┬──────────────────────────┤
│ 运行日志                      │ 运行摘要                  │
│ [10:31:02] 开始运行...        │ 成功：2                   │
│ [10:31:03] 第1步 成功         │ 失败：1                   │
│ [10:31:05] 第3步 失败         │ 跳过：0                   │
│ 截图：data/...                │ 总耗时：3.2s              │
├──────────────────────────────┴──────────────────────────┤
│ 当前步骤：第 3 步 点击登录 [查看截图] [查看HTML]          │
└─────────────────────────────────────────────────────────┘
```

### 5.2 进度显示

`widget.ProgressBar` 绑定到运行进度：
- `Max` = 当前流程启用步骤总数
- `Value` = 已完成的启用步骤数

进度条下方文本：`第 5 步 / 共 12 步 · 点击登录`

### 5.3 步骤状态可视化

步骤表格的「状态」列在运行时动态刷新：

| 状态 | 图标 | 颜色 |
|------|------|------|
| 未运行 | 空 | — |
| 运行中 | 蓝色圆点 | `#1a73e8` |
| 成功 | 绿色勾选 | `#4caf50` |
| 失败 | 红色叉号 | `#e53935` |
| 跳过 | 灰色横杠 | `#9e9e9e` |

> 注：Fyne `widget.Table` 单元格使用 `canvas.Text` + `canvas.Circle` 绘制状态图标，不使用动画（避免引入定时器复杂度）。

运行结束后 3 秒自动清除状态图标（或下次运行前清除）。

### 5.4 失败自动选中

`handleRunnerEvents()` 中，当收到 `EventStepDone` 且 `Result.Status == StatusFailed`：
1. 自动滚动步骤表格到失败行。
2. 选中失败步骤（`stepTable.Select(ev.StepIndex)`）。
3. 右侧属性区加载该步骤，方便立即修改 XPath 或输入。

### 5.5 日志产物链接

失败时日志格式：

```text
[10:31:05] 第 3 步 点击登录 失败：未找到 XPath //button[@id='login']
[10:31:05] 截图：data/run-history/2026-01-15/xxx/screenshot.png
[10:31:05] HTML：data/run-history/2026-01-15/xxx/page.html
```

路径使用纯文本（用户可复制），或在日志末尾放置 `widget.Hyperlink`（若 Fyne 支持打开文件）。

### 5.6 运行摘要面板

实时从 `RunResult` 更新：
- 成功数、失败数、跳过数
- 总耗时（毫秒转秒，保留 1 位小数）

## 6. 阶段 D：新手引导

### 6.1 首次启动检查

`initDeps()` 中增加 `firstRunCheck()`：

1. 检查 `chrome/` 下是否有 `chrome.exe`。
   - 无：状态栏显示 `未安装`，日志提示「未检测到 Chrome，请点击下载」。
2. 检查 `data/` 和 `logs/` 目录，不存在则创建。
3. 检查 `data/app-config.json`，不存在则生成默认配置。

### 6.2 空状态页

当 `flowStore.ListSorted()` 返回空时，中间区域显示空状态：

```text
┌──────────────────────────────┐
│        [文档图标]              │
│        暂无流程                │
│  点击左侧「新建」创建流程，     │
│  或导入示例流程快速体验。       │
│   [新建流程]  [导入示例]       │
└──────────────────────────────┘
```

使用 `widget.Card` 或 `container.NewCenter` 实现。

### 6.3 示例流程

新增 `flow.NewExampleLoginFlow()`：

| 步骤 | 操作类型 | 参数 |
|------|---------|------|
| 1 | 打开网址 | `https://example.com/login` |
| 2 | 输入文本 | XPath: `//input[@id='username']`, 输入: `${var:user=SP${11000-11099}}` |
| 3 | 输入文本 | XPath: `//input[@id='password']`, 输入: `Password123` |
| 4 | 点击元素 | XPath: `//button[@type='submit']` |
| 5 | 断言元素存在 | XPath: `//div[contains(text(),'欢迎')]` |
| 6 | 页面截图 | — |

流程名称：`示例：登录测试`，标签：`示例`。

点击「导入示例」时调用 `flowStore.Save(exampleFlow)` 并刷新列表。

## 7. 数据持久化不变性保证

- JSON schema **不变**：`flow.StepType`、`flow.ErrorPolicy`、`flow.Input` 结构保持原枚举。
- 中文只存在于 UI 展示层，`flow.Flow` 和 `flow.Step` 继续使用英文枚举。
- 新增字段（如 UI 状态、dirty 标志）均为内存状态，**不写入**流程 JSON。
- `flow.Validate()` 逻辑不变，UI 在保存前仍调用它做最终校验。
