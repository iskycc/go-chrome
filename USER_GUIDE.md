# Go Chrome 自动化编排工具 — 用户操作说明

## 1. 首次启动

双击 `go-chrome.exe` 启动程序。

**首次启动流程：**
1. 程序会自动创建 `data/`、`logs/`、`chrome/` 目录。
2. 如果 `chrome/` 目录中没有 `chrome.exe`，程序会自动从官方源下载 Chrome for Testing Stable 版本。
3. 下载完成后自动解压到 `chrome/` 目录。
4. 主窗口打开后，左侧显示流程列表（初始为空）。

> 首次下载需要联网，下载大小约 150MB。如果配置了自定义远程链接，会优先从该链接下载。

## 3. 创建自动化流程

### 3.1 新建流程

1. 点击工具栏「**新建流程**」。
2. 输入流程名称（如"登录后台"）。
3. 在「1. 流程属性」区域填写：
   - **名称**：流程的显示名称。
   - **描述**：流程的用途说明。
   - **标签**：用逗号分隔，如 `login, admin, daily`。

### 3.2 添加步骤

1. 在「2. 步骤编排」区域点击「**+ 新增**」。
2. 在「3. 步骤属性」区域编辑：
   - **步骤名称**：如"点击登录按钮"。
   - **操作类型**：从下拉菜单选择：
     - `打开网址` — 导航到指定 URL
     - `点击元素` — 点击 XPath 定位的元素
     - `输入文本` — 向输入框发送文本
     - `清空后输入` — 先清空再输入
     - `等待元素出现` — 等待 XPath 元素出现在 DOM 中
     - `等待元素可见` — 等待元素可见
     - `固定等待` — 等待固定毫秒数
     - `获取元素文本` — 读取元素文本内容
     - `断言元素存在` — 检查元素是否存在
     - `断言文本包含` — 检查元素文本是否包含指定内容
     - `页面截图` — 截取当前页面
   - **目标**：元素的 XPath，如 `//button[contains(., '登录')]`。
   - **输入内容**：仅对"输入文本"和"清空后输入"有效。
     - 支持模板表达式，如 `SP${11000-11099}`。
     - 点击「**预览**」可查看 3 个随机生成示例。
     - 点击「**校验**」检查模板语法。
   - **执行前等待(ms)**：执行该步骤前的等待时间。
   - **执行后等待(ms)**：执行该步骤后的等待时间（默认 500ms）。
   - **超时时间(ms)**：该步骤的最大执行时间（默认 10000ms）。
   - **失败处理**：
     - `失败后停止` — 失败时终止整个流程
     - `失败后继续` — 记录失败，继续执行后续步骤
     - `失败后重试` — 自动重试最多 2 次
   - **启用状态**：取消勾选可临时禁用该步骤。
   - **日志脱敏**：勾选后该步骤的输入值在日志中隐藏。

### 3.3 调整步骤顺序

- 选中步骤后，点击「**↑ 上移**」或「**↓ 下移**」调整顺序。
- 点击「**复制**」复制当前步骤。
- 点击「**删除**」删除当前步骤。

### 3.4 保存流程

点击工具栏「**保存**」。如果校验失败（如缺少必填字段），会弹出错误提示。

## 4. 执行流程

### 4.1 启动 Chrome

点击「4. 运行控制与日志」区域的「**启动浏览器**」，或工具栏的「**启动 Chrome**」。

- 如果本地没有 Chrome，会自动下载。
- Chrome 以无痕模式启动，使用独立的用户数据目录。

### 4.2 运行整个流程

1. 在左侧流程列表中选中要执行的流程。
2. 点击「**运行整个流程**」。
3. 执行过程中底部日志会实时显示每一步的状态。
4. 执行完成后，在「执行历史」区域可查看结果统计。

### 4.3 单步执行

1. 选中流程。
2. 点击「**单步执行**」按钮初始化 Chrome 环境。
3. 按钮变为「**下一步**」，每点击一次执行一个步骤。
4. 适合调试和观察每一步的效果。

### 4.4 从指定步骤运行

在步骤列表中选中某个步骤，然后点击「**运行整个流程**」，流程会从该步骤开始执行到结束。

### 4.5 停止执行

点击「**停止**」按钮可中断正在运行的流程。

## 5. 查看执行历史

在「2. 步骤编排」下方的「**执行历史**」区域：
- 显示当前流程的所有历史执行记录。
- 每条记录显示时间、状态、成功/失败步骤数。
- 历史记录保存在 `./data/run-history/` 目录中。
- 超过配置保留天数的记录会自动清理（默认 14 天）。

## 6. 导入和导出

### 6.1 导出流程

1. 选中要导出的流程。
2. 点击工具栏「**导出**」。
3. 选择保存路径，生成 `.json` 文件。

### 6.2 导入流程

1. 点击工具栏「**导入**」。
2. 选择之前导出的 `.json` 文件。
3. 导入的流程会自动分配新 ID，不会覆盖现有流程。

## 7. 输入模板使用指南

输入模板用于动态生成数据，支持以下占位符：

| 占位符 | 示例 | 说明 |
|--------|------|------|
| 数字范围 | `SP${11000-11099}` | 生成 11000~11099 的随机整数 |
| 固定位数 | `SHOP-${0001-9999}` | 生成 4 位数字，保留前导零 |
| 枚举随机 | `${dev\|test\|stage}` | 从候选值中随机选择一个 |
| 随机数字 | `${number:6}` | 6 位随机数字 |
| 随机字母 | `${alpha:8}` | 8 位随机字母 |
| 随机字母数字 | `${alnum:10}` | 10 位随机字母数字 |
| UUID | `${uuid}` | 生成标准 UUID |
| 时间戳 | `${timestamp}` | Unix 时间戳（秒） |
| 日期 | `${date:yyyyMMdd}` | 当前日期，如 20250605 |
| 日期时间 | `${datetime:yyyyMMddHHmmss}` | 当前日期时间 |
| 运行序号 | `${seq}` | 同一次执行中按步骤递增 |
| 变量复用 | `${var:user=SP${11000-11099}}` | 首次生成并保存变量 |
| 变量读取 | `${var:user}` | 读取同一次执行中已保存的变量 |

**组合示例：**
- 业务编号：`ORD-${date:yyyyMMdd}-${number:6}` → `ORD-20250605-482910`
- 手机号：`138${00000000-99999999}` → `13804293827`
- 邮箱：`${alnum:8}@example.com` → `a3B9xK2m@example.com`

## 8. 配置说明

配置文件位于 `./data/app-config.json`：

```json
{
  "chrome": {
    "downloadSource": "official_stable",
    "customDownloadURL": "",
    "customDownloadSHA256": "",
    "fallbackToOfficial": true,
    "installDir": "./chrome",
    "userDataDir": "./data/chrome-profile",
    "keepDownloadCache": true
  },
  "runner": {
    "defaultTimeoutMs": 10000,
    "defaultWaitAfterMs": 500,
    "templatePreviewCount": 5,
    "maskInputValueInLogs": true,
    "autoScreenshotOnFinish": false
  },
  "app": {
    "closeManagedChromeOnExit": true,
    "logRetentionDays": 14,
    "theme": "default"
  }
}
```

修改配置后重启程序生效。

## 9. 常见问题速查

| 问题 | 解决方法 |
|------|---------|
| 首次启动下载很慢 | 配置自定义下载源或使用离线预置包 |
| Chrome 启动失败 | 检查是否有其他程序占用端口；删除 `./data/chrome-profile` 后重试 |
| XPath 找不到元素 | 使用「等待元素出现」步骤增加等待时间；检查页面是否加载完成 |
| 输入模板预览报错 | 检查模板语法，确保 `${}` 成对出现 |
| 流程保存失败 | 检查所有步骤是否有名称，需要目标的操作是否填写了 XPath |
| 日志显示输入值 | 勾选步骤的「日志脱敏」选项 |

## 10. 离线使用

1. 在有网络的环境首次启动，等待 Chrome 下载完成。
2. 将整个项目目录（含 `./chrome`）复制到离线环境。
3. 离线环境直接运行 `go-chrome.exe`，无需联网。

## 11. 数据备份

定期备份以下目录：
- `./data/flows/` — 所有自动化流程
- `./data/app-config.json` — 应用配置

流程文件是纯 JSON，可直接复制、版本控制或跨环境迁移。
